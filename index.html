Windows NT 4.0

Windows NT 4.0 is a major release of the Windows NT operating system developed by Microsoft, targeting the data server and personal workstation markets. It was released to manufacturing on July 31, 1996, and to retail on August 24, 1996.

Its most prominent user-facing change was the adoption of the Windows 95 user interface — bringing the Start menu, taskbar, and Windows Explorer shell to the enterprise-grade NT kernel for the first time. Internally, Microsoft referred to it as the Shell Update Release (SUR).

NT 4.0 was the direct successor to Windows NT 3.51, and was itself succeeded by Windows 2000 in December 1999. It was also the last version of Windows NT to carry the “NT” branding in its product name — Windows 2000, while built on NT technology, dropped the moniker.

Despite Microsoft's push to migrate customers to Windows 2000, NT 4.0 remained widely deployed in enterprises well into the mid-2000s, testament to its stability and reliability.

Sources: Wikipedia, BetaWiki

History

Background

Windows NT 3.51 used the older Program Manager interface inherited from Windows 3.x. Meanwhile, Windows 95 had debuted with its revolutionary new shell. Microsoft's goal with NT 4.0 was to bring that familiar UI to the NT kernel — creating an OS suitable for both desktop users and corporate servers.

Development

The project was codenamed Shell Update Release (SUR), sometimes also referred to as Tukwila. A key architectural decision was to move the graphics subsystem (GDI) from user mode into kernel mode. This dramatically improved graphics performance but meant a buggy driver could destabilise the entire system.

Release

NT 4.0 was released to manufacturing on July 31, 1996, followed by retail availability on August 24, 1996. Server editions followed in early September 1996. The release was well received, particularly in corporate environments.

Legacy

NT 4.0 became the longest-running version of Windows NT until Windows 2000's release. Over its lifecycle it received six official service packs. A planned seventh service pack was cancelled in April 2001.

On 12 February 2004, over 30,000 source files for Windows NT 4.0 were leaked onto the internet. The leak was later traced back to Mainsoft.

Sources: Wikipedia, BetaWiki

Features

Shell & UI

  • Start menu and taskbar
  • Windows NT Explorer
  • “My Computer”, “My Documents” shell folders
  • Recycle Bin, shortcut icons, desktop as a folder
  • Per-user Start menu shortcuts

Extras from Microsoft Plus!

  • Space Cadet Pinball (3D Pinball)
  • Font smoothing (anti-aliasing)
  • Full window drag
  • High-colour icons
  • Wallpaper stretch-to-fit

New Technical Components

  • CryptoAPI — Microsoft's cryptography API, introduced here
  • DCOM — Distributed Component Object Model
  • TAPI 2.0 — Telephony API, first TAPI release on Windows NT
  • Task Manager — the familiar Ctrl+Alt+Del task switcher
  • Winsock 2 and improved TCP/IP stack
  • DirectX support (limited)
  • Internet Explorer 2 bundled
  • Partial Bluetooth 1.0 support (Service Pack 5+)

GDI in Kernel Mode

Moving the GDI into kernel mode yielded a major performance boost but at the cost of stability: a buggy display driver could crash the entire OS.

Sources: Wikipedia, BetaWiki

Editions

Workstation

The standard desktop edition for business users. Supported up to 2 processors and 4 GB of RAM.

Server

Included IIS 2.0, Microsoft FrontPage 1.1, NetShow Services, and Remote Access Service.

Enterprise Edition

Supported up to 8 processors and clustering via Microsoft Cluster Server (MSCS).

Terminal Server Edition

Added multi-user remote desktop capabilities — a precursor to Remote Desktop Services.

Embedded

A modular variant for embedded systems. Extended support lasted until July 11, 2006.

Sources: Wikipedia, BetaWiki

Hardware

Supported Architectures

  • x86 (IA-32) — PC/AT and PC-98 compatible systems
  • DEC Alpha — supported until August 1999
  • MIPS — only SP1 was released for MIPS
  • PowerPC — SP2 was the final service pack for PowerPC

NT 4.0 was the last version of Windows to support Alpha, MIPS, and PowerPC.

x86 System Requirements

  • 486 processor or better
  • Minimum 16 MB RAM
  • 110 MB hard drive space
  • VGA or better display
  • CD-ROM drive

Known Compatibility Issue

Early builds would crash if run on a processor with a CPUID value higher than 3. Fixed in Service Pack 6.

Sources: BetaWiki, Wikipedia

Trivia & Easter Eggs

The Hidden Credits Easter Egg

Create a folder on the desktop and rename it sequentially to:

  1. and now, the moment you've all been waiting for
  2. we proudly present for your viewing pleasure
  3. The Microsoft Windows NT Product Team!

Implemented in SHELL32.DLL, folder names hashed and XOR'd with 0x95. Early builds referenced the "SUR Shell Team" codename.

The Source Code Leak

On February 12, 2004, over 30,000 source files for Windows NT 4.0 were leaked online. The leak was traced back to Mainsoft, a company that had licensed the source code to port Windows APIs to Unix.

The Cancelled SP7

SP7 was cancelled in April 2001. The Security Rollup Update (SP6a SRU) was released in its place, signalling Microsoft's push to move customers to Windows 2000 and XP.

Space Cadet Pinball

NT 4.0 was one of the first Windows versions to ship with 3D Pinball: Space Cadet, sourced from Maxis' Full Tilt! Pinball. It survived through Windows XP before being dropped in Vista.

Sources: BetaWiki, Wikipedia
